# Contributing to BlogApp

Thank you for your interest in contributing to the BlogApp! We welcome contributions from developers of all skill levels.

## How to Contribute

### 1. Report Bugs
- Search existing issues before opening a new one.
- Provide a clear and descriptive title.
- Include steps to reproduce the bug.
- Mention your OS and Node.js version.

### 2. Suggest Enhancements
- Explain why the enhancement would be useful.
- Provide a clear and descriptive title.

### 3. Submit Pull Requests
1. Fork the repository.
2. Create a new branch for your feature or bugfix: `git checkout -b feature/your-feature-name`.
3. Follow the project's coding standards (use Prettier for formatting).
4. Write clear commit messages.
5. Ensure your code doesn't break existing functionality.
6. Submit a pull request to the `main` branch.

## Project Structure
- `Backend/`: The Express.js server and API logic.
- `BlogAppFrontend/`: The React.js frontend built with Vite.
- `docs/`: Project documentation.

## Development Setup

1. Install all dependencies:
   ```bash
   npm run install:all
   ```
2. Start development servers:
   ```bash
   npm run dev
   ```

## Coding Standards
- Use ES6 modules (import/export).
- Keep functions small and modular.
- Use descriptive naming for variables and functions.
- Run Prettier to ensure consistent formatting.
